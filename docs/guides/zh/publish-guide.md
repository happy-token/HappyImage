# HappyImage 发布指南

多平台一键发布的使用说明和常见问题。

## 平台规则速查

| 平台 | 最多图片 | 标题限制 | 正文限制 | 话题标签 |
| :--- | :---: | :--- | :--- | :--- |
| **小红书** | 18 | 20 字 | 1000 字 | 最多 10 个，正文末尾单独一行 |
| **微博** | 18 | 内联【标题】 | 2000 字（140 字折叠） | `#话题#` 格式内联在正文中 |
| **X (Twitter)** | 4 | 无独立标题 | 280 字符 | `#tag` 格式内联在正文中 |
| **微信公众号** | 不限 | 64 字 | 20000 字 | **不支持话题标签** |

## 发布流程

1. 生成图片和文案后，在项目页切换到"发布"标签
2. 选择目标平台
3. 点击"生成配文"，AI 按照平台格式要求生成文案
4. 在平台预览中检查效果，可直接点击文案编辑
5. 点击"自动填写发布"，Chrome 浏览器自动打开并填入内容
6. **手动核对**：工具不会自动点击发布按钮，请在浏览器中确认后手动发布

## 登录态管理

自动发布依赖本地 Chrome 浏览器的登录态。所有平台共享同一个 Chrome Profile。

- 首次使用：在设置页点击各平台的扫码登录按钮，扫码后会话自动保存
- 后续使用：无需重复登录，Chrome 自动复用已保存的登录态
- 存储路径：`~/Library/Application Support/HappyImage/chrome-profile` (macOS)

登录态过期时，浏览器顶部会显示黄色提示条，重新扫码后自动继续。

## 各平台特点

### 小红书
- 发布页：`creator.xiaohongshu.com/publish/publish`
- 自动切换至"上传图文"选项卡
- 首图建议 3:4 竖版

### 微博
- 发布页：`weibo.com`
- 最多 18 张图片
- 3 图布局：1+2 特殊排列
- 支持头条文章长文模式

### X (Twitter)
- 发布页：`x.com/compose/post`
- 最多 4 张图片
- 防检测剪贴板模拟粘贴方式上传
- 支持 X Articles 长文（需 Premium）

### 微信公众号
- 发布页：`mp.weixin.com`
- 支持 API（快速）和浏览器两种发布方式
- 自动将 Markdown 转化为微信富文本格式
- 支持多账号管理（通过别名切换）

## 常见排查

**Q: 浏览器弹出了，但文案和图片没有填进去？**
A: 账号登录过期。检查浏览器顶部是否有黄色提示条，如有请重新扫码登录。

**Q: 自动填充时输入框有残留旧文案？**
A: 工具在填写前会执行清空指令，但部分富文本编辑器有缓存。如发现重叠，手动全选清除。

**Q: 微信账号别名怎么配置？**
A: 在设置页 → 平台发布 → 公众号账号向导中录入多个公众号，发布面板中可切换。

**Q: 平台预览显示话题标签数量为“内联”？**
A: 微博和 X 的话题标签是正文的一部分，不是独立字段。“内联”表示会被计入正文字符数。

**Q: 如何配置图片水印？**
A: 在设置页 → 偏好设置 → Skill 偏好中选择对应 Skill，编辑水印内容、位置、透明度。
