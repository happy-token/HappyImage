设置数据存在哪里？

API Key 和用户偏好的存储方式决定架构。这是个本地单用户工具。

A

localStorage（纯前端）

存在浏览器本地存储。无需后端改动，但换浏览器或清除缓存就丢失。API Key 明文存储在浏览器中。

优点

  • 零后端改动
  • 即时生效

缺点

  • 不跨会话/浏览器
  • 浏览器清理即丢
B

服务端 JSON 文件（推荐）

存在 ~/.baoyu-skills/config.json，后端读写。重启不丢、跨浏览器，和其他 CLI 工具共享配置。

优点

  • 持久化，重启不丢
  • 跨浏览器共享
  • 和 CLI 同一套 Key

缺点

  • 需要后端新增 API
C

服务端 .env 文件

直接写入项目 .env，和其他工具共享同一套环境变量。

优点

  • 与现有 env 体系一致

缺点

  • 修改后需重启服务器
  • 容易误入 git 提交
